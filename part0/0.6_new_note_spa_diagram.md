# Exercise 0.5: Single Page App (SPA) diagram

## Here Mermaid Syntax

    sequenceDiagram
        participant browser
        participant server

        browser->>server: GET [https://studies.cs.helsinki.fi/exampleapp/spa]
        activate server
        server-->>browser: HTML document
        deactivate server

        browser->>server: GET [https://studies.cs.helsinki.fi/exampleapp/main.css]
        activate server
        server-->>browser: the CSS file
        deactivate server

        browser->>server: GET [https://studies.cs.helsinki.fi/exampleapp/spa.js]
        activate server
        server-->>browser: the JavaScript file (spa.js)
        deactivate server

        Note right of browser: The browser starts executing the spa.js file which fetches the JSON data

        browser->>server: GET [https://studies.cs.helsinki.fi/exampleapp/data.json]
        

        Note right of browser: The browser executes the callback function that renders the notes to the page dynamically

## Here Mermaid Diagram

```mermaid
sequenceDiagram
    participant browser
    participant server

    Note over browser: The user writes a note in the text field and clicks the Save button
    Note over browser: The JS code (spa.js) intercepts the submit event, adds the note to the local list, and redraws the DOM

    browser->>server: POST https://studies.cs.helsinki.fi/exampleapp/new_note_spa
    activate server
    Note over browser: Send the new note as JSON data: { "content": "SPA note", "date": "2026-07-16" }
    server-->>browser: HTTP Status Code 201 Created (JSON: {"message":"note created"})
    deactivate server

    Note over browser: No page redirect or reload happens! The UI is already updated locally.
