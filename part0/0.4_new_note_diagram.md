# Exercises 0.4: New note diagram

## Here Mermaid Syntax

    sequenceDiagram
        participant browser
        participant server

        Note over browser: The user writes a note in the text field and clicks the Save button

        browser->>server: POST https://studies.cs.helsinki.fi/exampleapp/new_note
        activate server
        Note over server: The server extracts the data, saves it to the database/array, and prepares a redirect
        server-->>browser: HTTP status code 302 (Redirect to /exampleapp/notes)
        deactivate server

        Note over browser: The redirect triggers the browser to reload the notes page automatically

        browser->>server: GET https://studies.cs.helsinki.fi/exampleapp/notes
        activate server
        server-->>browser: HTML document
        deactivate server

        browser->>server: GET https://studies.cs.helsinki.fi/exampleapp/main.css
        activate server
        server-->>browser: the CSS file
        deactivate server

        browser->>server: GET https://studies.cs.helsinki.fi/exampleapp/main.js
        activate server
        server-->>browser: the JavaScript file
        deactivate server

        Note right of browser: The browser starts executing the JavaScript code that fetches the JSON from the server

        browser->>server: GET https://studies.cs.helsinki.fi/exampleapp/data.json
        activate server
        server-->>browser: [{ "content": "Your new note", "date": "2026-07-16" }, ... ]
        deactivate server

        Note right of browser: The browser executes the callback function that renders all notes (including the new one)
